Integration Guide

Step-by-step guides to help you integrate our API into your applications.

Web Application Integration

Integrating our API into a web application is straightforward. Here's a complete example using JavaScript and modern web technologies.

Setting Up Authentication

First, create a secure environment for your API key. Never expose API keys in client-side code.

// config.js - Store this securely on your server
const API_CONFIG = {
  baseURL: 'https://api.oscarrondon.com/v1',
  apiKey: process.env.API_KEY,
  timeout: 10000
};

class ApiClient {
  constructor(config) {
    this.baseURL = config.baseURL;
    this.apiKey = config.apiKey;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseURL}${endpoint}`;
    const config = {
      headers: { 'Authorization': `Bearer ${this.apiKey}`, 'Content-Type': 'application/json', ...options.headers },
      ...options
    };
    const response = await fetch(url, config);
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return await response.json();
  }
}

Mobile Application Integration

Our API works seamlessly with mobile applications. Here are best practices for iOS and Android integration.

// iOS Swift Example
class APIService {
    private let baseURL = "https://api.oscarrondon.com/v1"
    private let apiKey: String

    func fetchContent(completion: @escaping (Result<[Content], APIError>) -> Void) {
        guard let url = URL(string: "\(baseURL)/content") else {
            completion(.failure(.invalidURL)); return
        }
        var request = URLRequest(url: url)
        request.setValue("Bearer \(apiKey)", forHTTPHeaderField: "Authorization")
        URLSession.shared.dataTask(with: request) { data, _, error in
            // handle response
        }.resume()
    }
}

Server-to-Server Integration

For backend services, implement caching and rate limiting to optimize performance.

import requests
from functools import lru_cache

class APIClient:
    def __init__(self, api_key: str):
        self.session = requests.Session()
        self.session.headers.update({'Authorization': f'Bearer {api_key}'})
        self.base_url = "https://api.oscarrondon.com/v1"

    @lru_cache(maxsize=100)
    def get_content(self, content_type=None):
        params = {'type': content_type} if content_type else {}
        response = self.session.get(f"{self.base_url}/content", params=params)
        response.raise_for_status()
        return response.json()['data']

Best Practices

Follow these recommendations to build robust, scalable applications.

Authentication & Security

API Key Security

  • Never expose API keys in client-side code
  • Use environment variables for server applications
  • Rotate API keys regularly
  • Implement IP whitelisting when possible

Request Security

  • Always use HTTPS for API requests
  • Validate and sanitize user inputs
  • Implement proper error handling
  • Log security events for monitoring

Performance Optimization

Optimize your API usage for better performance and cost efficiency.

// Good: Use pagination for large datasets
const fetchAllContent = async () => {
  let allContent = [];
  let offset = 0;
  const limit = 50;
  while (true) {
    const response = await apiClient.request(`/content?limit=${limit}&offset=${offset}`);
    allContent.push(...response.data);
    if (!response.pagination.has_more) break;
    offset += limit;
  }
  return allContent;
};

Error Handling Patterns

class APIError extends Error {
  constructor(message, status, code) {
    super(message);
    this.status = status;
    this.code = code;
  }
}

async function requestWithRetry(endpoint, maxRetries = 3) {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      const response = await fetch(endpoint, { headers: getHeaders() });
      if (response.status === 429) {
        const wait = response.headers.get('Retry-After') || Math.pow(2, attempt);
        await new Promise(r => setTimeout(r, wait * 1000));
        continue;
      }
      if (!response.ok) throw new APIError('Request failed', response.status, 'ERR');
      return await response.json();
    } catch (e) {
      if (attempt === maxRetries - 1) throw e;
    }
  }
}

Troubleshooting

Common issues and their solutions to help you resolve problems quickly.

Common Errors

401 Unauthorized Authentication Failed

Cause: Invalid or missing API key

  • Verify your API key is correct and active
  • Check that the Authorization header is properly formatted
  • Ensure your API key has the necessary permissions
429 Rate Limited Too Many Requests

Cause: Exceeded rate limits

  • Implement exponential backoff retry logic
  • Respect the Retry-After header
  • Implement request queuing and batching
